Genau, und dann abschließend kann man auch Arrays von Objekten haben, weil wir haben
Arrays gesehen, Strings haben wir auch gesehen, das sind eigentlich, oder ein String würde
definiert als eine Klasse, deshalb jedes Mal, wenn wir ein String benutzen, haben wir eigentlich
ein Objekt von Klasse String und wie wir auch gesehen haben, Array kann man nicht nur von
Integers oder Doubles haben, man kann auch ein Array haben von Objekten.
Das heißt, wenn wir zum Beispiel unserem Snail-Beispiel, unserem Spiel, nicht nur einen
Feind oder Gegner haben möchten, aber gleich vier, die wir in alle Ecke stecken, dann können
wir die Gegner als ein Array implementieren.
Weil wir haben schon einen Gegner, wir können genauso ein Array von Gegner, von Enemies
in diesem Falle, konstruieren.
Und das kann man so machen, also grundsätzlich kann man erst sagen, wir möchten jetzt ein
neuer Array konstruieren in unserem Speich mit Größe 2 und erst nachher muss man dann
für jedes Element von diesem Array, weil das sind Objekte, muss man diese Objekte
auch erst noch kreieren.
Das heißt, wenn man das hier, diese erste Zeile, hat man ein Array im Speicher reserviert,
aber dieser Array hat eigentlich zwei Referenzvariablen, die irgendwo anders im Speicher zeigen.
Und am Anfang zeigen die nirgendwo hin, also sind die Objekte noch nicht kreiert.
Deshalb muss man dann mit diesem Iterator, also das A1.0 und A1.1, das sind die zwei
Werte in unserem Array, die muss man erst auch noch kreieren im Speicher.
Das heißt, erst wenn man New Student macht, hat man ein neues Objekt von Typ Student kreiert
mit diesem Inhalt oder diesen Konstruktoren hier in diesem Fall.
Das kann man etwas schneller machen, indem man einfach mit den geschweiften Klammern
auf und zu genau das gleiche in einem macht.
Natürlich ist das auch schwieriger, je größer dieses Array wird.
Also meistens, wenn dieses Array relativ groß wird, macht man das mit einer Vorschleife
zum Beispiel, wo man jedes Element in diesem Array erst mal reservieren soll im Speicher
mit unserem Stichwort New Student.
