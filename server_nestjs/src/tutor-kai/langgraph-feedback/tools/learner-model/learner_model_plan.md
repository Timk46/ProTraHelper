# Plan: Learner Model as a Langchain Tool

## 1. Goal
Enhance the feedback system's adaptivity by providing Langgraph agents with access to a richer Learner Model via a dedicated Langchain tool. This tool will fetch structured data and generate a task-specific textual summary on demand.

## 2. Learner Model DTO (`LearnerModelDto`)
The data structure returned by the tool's underlying service:

### Structured Fields:
*   `performanceOnCurrentConcept`: `{ attempts: number, avgScore: number | null, commonErrorTypes: string[], lastFeedbackTypes: string[] } | null`
*   `performanceOnPrerequisites`: `{ conceptId: number, conceptName: string, level: number | null, recentStruggles: boolean }[]`
*   `overallCodingPerformance`: `{ totalSubmissions: number, overallAvgScore: number | null }`
*   `recentActivity`: `{ lastSubmissionDate: Date | null, timeSinceLastAttemptSeconds: number | null }` (Note: `timeSinceLastAttemptSeconds` might be null/less accurate in the tool context)

### Summary Field:
*   `taskHistorySummary: string | null`: A concise narrative summary of the student's interaction with the *specific task* being queried.

## 3. `taskHistorySummary` Content Definition
The summary generated by the service should capture:
*   Number of prior attempts on this task.
*   Error trajectory (types of errors, evolution/persistence).
*   Feedback sequence (types of feedback received: KM, KH, KC, KTC, __END__).
*   Inferred responsiveness to previous feedback on this task.

## 4. Implementation Steps (Refactored Tool Approach)
*   **Create Tool Directory:** `server_nestjs/src/tutor-kai/langgraph-feedback/tools/learner-model/`.
*   **Implement Service (`learner-model.service.ts`):** Create `LearnerModelToolService` within the tool directory.
    *   Encapsulate Prisma queries for overall performance, task history, concept performance, and prerequisite performance.
    *   Include Langchain logic (`ChatOpenAI`) to generate `taskHistorySummary`.
    *   The main method (e.g., `getLearnerModelData`) takes `userId`, `codingQuestionId`, `conceptNodeId` and returns the `LearnerModelDto`.
*   **Implement Tool (`learner-model.tool.ts`):** Define a `DynamicStructuredTool` named `fetch_learner_model`.
    *   Define input schema (Zod): `userId`, `codingQuestionId`, `conceptNodeId` (nullable).
    *   Provide a clear description for the LLM agent.
    *   The tool's function calls `LearnerModelToolService.getLearnerModelData` and returns the result as a JSON string.
*   **Cleanup:**
    *   Delete the old `helper/learner-model.service.ts`.
    *   Revert changes made to `helper/langgraph-data-fetcher.service.ts` (remove service injection/calls).
    *   Revert changes made to `shared/dtos/tutorKaiDtos/FeedbackContext.dto.ts` (remove `learnerModel` property).
    *   Revert changes made to `agents/supervisor/supervisor.prompt.ts` (remove direct references to `learnerModel` object in context/reasoning).
*   **Integration (Future Step):** Add the `fetch_learner_model` tool to the appropriate Langgraph agent's tool list.
*   **Prompt Update (Future Step):** Update the agent's prompt to instruct it on *when* and *how* to use the `fetch_learner_model` tool and interpret its JSON string result.
